# Authorized Capture Workflow

This workflow is for interoperability, installer planning and validation on software, accounts and devices the operator is authorized to use.

## Naming rule

The session family is always a TTG family such as `ttg-meta`, `ttg-adb`, `ttg-fastboot` or `ttg-qualcomm`.

The external application being examined belongs only in `observed_product`. For example, TSM may be recorded as the observed product for one authorized capture, but it must not be used as our tool-family, engine, folder or runtime name.

## 1. Prepare the workstation

- Use an isolated Windows test machine or VM where practical.
- Create a restore point or snapshot.
- Record the Windows edition/build and architecture without storing the hostname, user name or machine ID.
- Install only prerequisites supplied by official vendors or already approved by THETECHGUY.
- Use a dedicated test device with no customer data.

## 2. Start a session

Run:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/New-CaptureSession.ps1 `
  -ToolFamily "ttg-meta" `
  -ProductName "Observed external product" `
  -ProductVersion "x.y.z"
```

For an authorized TSM observation, `ProductName` may identify TSM because that field records the external source. The created folder and `tool_family` still remain `ttg-meta`.

The script creates a session folder and a starter manifest. Keep raw working material outside the Git repository until it is sanitized.

## 3. Capture the operating contract

Record evidence in these groups:

### Files and runtimes

- executable/DLL/driver names;
- file and product versions;
- architecture;
- SHA-256;
- Authenticode status and signer;
- Visual C++, .NET, Java, Python or other runtime requirements;
- required services, scheduled tasks and environment settings;
- vendor source and installer order.

### USB and device modes

- VID/PID and interface class for each observed mode;
- driver provider, version and INF name;
- transition sequence such as Preloader -> META -> ADB;
- timeout/retry behaviour;
- whether reconnect, reboot or power cycle is required.

Do not retain a device serial number or other unique identifier. Reduce USB identifiers to stable VID/PID/interface facts.

### Operations

For each operation record:

- preconditions;
- selected device/profile;
- observable request stage names;
- expected mode and transport;
- sanitized result and error category;
- state change confirmed on the test device;
- backup/rollback requirement;
- whether the operation is read-only, state-changing or destructive.

Prefer evidence generated by our own wrapper/logging layer. Never capture credentials or bypass licensing controls.

### Network/source behaviour

Record only what is needed for lawful installation or interoperability:

- official vendor download page or API reference;
- domain and route purpose;
- expected content type and checksum source;
- offline fallback behaviour.

Do not commit authorization headers, cookies, signed temporary URLs, access tokens or account/session material.

## 4. Inventory a directory

```powershell
powershell -ExecutionPolicy Bypass -File scripts/Collect-FileInventory.ps1 `
  -TargetPath "C:\Approved\ObservedTool" `
  -SessionDirectory "sessions\ttg-meta\20260721T150000Z-ttg-meta-observed-product"
```

The collector writes metadata only. It does not copy the target files.

## 5. Classify redistribution

Every dependency must be classified as one of:

- `redistributable` — licence/source explicitly permits packaging;
- `vendor_fetch` — installer may fetch from the official vendor source;
- `operator_supplied` — licensed operator must supply it locally;
- `metadata_only` — evidence may be retained but the artifact must never be distributed;
- `unknown_blocked` — do not promote until reviewed.

See `REDISTRIBUTION_POLICY.md`.

## 6. Sanitize before commit

Before committing a session:

- remove raw logs and packet captures;
- replace absolute paths with relative paths;
- remove usernames, hostnames and machine identifiers;
- remove IMEI, serial, ECID, UDID, phone numbers and account IDs;
- remove tokens, cookies, URLs containing signatures and credentials;
- confirm no proprietary binary was staged;
- run `python scripts/validate_intake.py`.

## 7. Promotion

A reviewed intake record can later be transformed into the Software Builder TTG package contract:

- exact TTG tool/dependency ID;
- version and architecture;
- approved source;
- file name and expected size;
- SHA-256;
- install/detection rule;
- channel and rollback rule;
- redistribution class.

Missing, contradictory, externally branded or unlicensed internal metadata blocks promotion.
